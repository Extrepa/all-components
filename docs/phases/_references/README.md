# Phase Reference Files

**Purpose:** Reference documentation for phase work standards, checklists, and workflows.

This directory contains reusable reference files that document standards, checklists, and workflows for different types of phase work. Use these as guides when creating new phase documentation.

## Available References

### [Build Verification Checklist](BUILD_VERIFICATION_CHECKLIST.md)
Standard checklist for verifying builds across all project types (Vite, Next.js, Electron, etc.). Includes common issues and fixes.

**Use when:** Creating build verification phase documentation.

### [Code Quality Standards](CODE_QUALITY_STANDARDS.md)
Standards and review criteria for code quality. Covers linting, TypeScript, code style, organization, and best practices.

**Use when:** Creating code quality review phase documentation.

### [Test Coverage Standards](TEST_COVERAGE_STANDARDS.md)
Standards and goals for test coverage. Includes coverage goals by project category, test types, and infrastructure setup.

**Use when:** Creating test coverage phase documentation.

### [Feature Implementation Workflow](FEATURE_IMPLEMENTATION_WORKFLOW.md)
Standard workflow for implementing features. Covers planning, implementation, testing, documentation, review, and deployment phases.

**Use when:** Creating feature implementation phase documentation.

### [Documentation Standards](DOCUMENTATION_STANDARDS.md)
Standards and requirements for project documentation. Covers required files, quality standards, and maintenance.

**Use when:** Creating documentation phase documentation.

### [Production Readiness Checklist](PRODUCTION_READINESS_CHECKLIST.md)
Comprehensive checklist for production deployment. Covers pre-deployment, deployment, and post-deployment checks.

**Use when:** Creating production readiness phase documentation.

## Using These References

1. **Review relevant reference** before starting phase work
2. **Use as checklist** to ensure completeness
3. **Customize as needed** for specific project requirements
4. **Update references** based on lessons learned

## Workflow

1. **Choose template** from `_Resources/_Templates/Phases/`
2. **Copy template** to appropriate phase subdirectory
3. **Review relevant reference** from this directory
4. **Fill in project-specific information**
5. **Follow standards and checklists** from references
6. **Update as work progresses**
