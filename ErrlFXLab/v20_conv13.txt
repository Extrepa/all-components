1. Wire audioReactiveValue into the p5 sketch

You already have a p5 setup like:

function startP5WithCurrentSvg() {
  // ... create new p5((p) => { ... }, p5Host);
}

Inside that p5 sketch you’ll have a p.draw = function () { ... } with some wobble logic using p5Config.wobbleAmplitude, p5Config.wobbleSpeed, etc.

1.1 Add an audio boost helper inside the p5 sketch

Inside the p5 constructor (the new p5((p) => { ... }) callback), add this small helper near the top:

      function getAudioBoost() {
        // global audioReactiveValue comes from outer script
        if (!audioReactiveEnabled) return 1;
        return 1 + (audioReactiveValue || 0) * 0.9;
      }

Now anywhere you compute wobble or glow, you can modulate with getAudioBoost().

1.2 Apply audio boost to wobble and glow

Find where you compute your main wobble (examples):

      const t = p.millis() / 1000;
      const wobblePhase = t * p5Config.wobbleSpeed;
      const wobbleRadius = p5Config.wobbleAmplitude;
      const wobbleX = Math.sin(wobblePhase) * wobbleRadius;
      const wobbleY = Math.cos(wobblePhase * 0.7) * wobbleRadius;

Change to:

      const t = p.millis() / 1000;
      const audioBoost = getAudioBoost();

      const wobblePhase = t * p5Config.wobbleSpeed * audioBoost;
      const wobbleRadius = p5Config.wobbleAmplitude * audioBoost;
      const wobbleX = Math.sin(wobblePhase) * wobbleRadius;
      const wobbleY = Math.cos(wobblePhase * 0.7) * wobbleRadius;

And if you have any glow / bloom strength:

      const glow = p5Config.glowStrength;

Make it:

      const glow = p5Config.glowStrength * (audioReactiveEnabled ? 1 + audioReactiveValue * 0.5 : 1);

Now:
	•	Quiet audio → normal wobble / glow
	•	Loud audio → faster phase + bigger wobble radius + brighter glow

No config changes needed; it just rides on top of your existing presets.

2.2 Hook up Help overlay behavior

In your JS, grab the new elements:

    const helpOverlayBtn = document.getElementById("helpOverlayBtn");
    const helpOverlay = document.getElementById("helpOverlay");
    const helpOverlayCloseBtn = document.getElementById("helpOverlayCloseBtn");

Then add small helpers:

    function openHelpOverlay() {
      if (!helpOverlay) return;
      helpOverlay.classList.remove("hidden");
      playBeep(720, 0.07, "triangle");
    }

    function closeHelpOverlay() {
      if (!helpOverlay) return;
      helpOverlay.classList.add("hidden");
      playBeep(320, 0.07, "square");
    }

    helpOverlayBtn.addEventListener("click", () => {
      openHelpOverlay();
    });

    helpOverlayCloseBtn.addEventListener("click", () => {
      closeHelpOverlay();
    });

    if (helpOverlay) {
      helpOverlay.addEventListener("click", (e) => {
        if (e.target === helpOverlay) {
          // Clicked backdrop, not card
          closeHelpOverlay();
        }
      });
    }

2.3 Add keyboard shortcut for Help

In your global window.addEventListener("keydown", ...) handler, we already mapped keys like q/w/e/r/t, g, v, space, 1/2, c, s.

Add an h shortcut near the top:

      const key = event.key.toLowerCase();

      // Help overlay
      if (key === "h") {
        openHelpOverlay();
        return;
      }

And also make Esc close the overlay if it’s open:

      if (event.key === "Escape") {
        if (!helpOverlay.classList.contains("hidden")) {
          closeHelpOverlay();
          return;
        }
      }

Make sure those sit before other key handling so they fire first.
