# Project Integration Checklist

**Quick Reference:** Use this checklist after creating project documentation  
**Full Guide:** [Project-Integration-Guide](Project-Integration-Guide.md) - Complete integration guide

## Code Documentation (Required)

- [ ] README.md (use `_Resources/_Templates/README.md.template`)
- [ ] INDEX.md (use `_Resources/_Templates/INDEX.md.template`)
- [ ] PROJECT_STATUS.md (use `_Resources/_Templates/PROJECT_STATUS.md.template`)
- [ ] docs/index.md (use `_Resources/_Templates/Documentation/Core/docs-index.md.template`)

## Technical Documentation (Recommended)

- [ ] docs/architecture.md (if complex, use template)
- [ ] docs/project-structure.md (if organized, use template)
- [ ] docs/DEVELOPER_GUIDE.md (use template)

## Workspace Integration (Required)

- [ ] Added to PROJECTS_DASHBOARD.md
  - [ ] Appropriate category selected (A/B/C/D)
  - [ ] Status tag added
  - [ ] Code location path added: `/Users/extrepa/Projects/project-name`
  - [ ] Links added to README and docs

## Optional Project Management Docs

If using separate project management documentation:

- [ ] Created folder in `_Resources/Vault/Projects/Project-Name/`
- [ ] README.md (project management overview)
- [ ] specs.md (technical specifications)
- [ ] tasks.md (task tracking)
- [ ] versions.md (version history)
- [ ] prompts.md (AI prompts)
- [ ] build-order.md (build sequence)
- [ ] assets.md (asset documentation)

## Cross-References

- [ ] Code README links to management docs (if applicable)
- [ ] Management docs link to code location
- [ ] Related projects cross-referenced

## Verification

- [ ] Folder name follows convention (lowercase, hyphenated for code)
- [ ] All links work correctly
- [ ] Documentation follows template structure
- [ ] No broken references
