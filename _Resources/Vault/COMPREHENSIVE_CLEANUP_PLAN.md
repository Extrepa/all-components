## Naming Conventions (Finalized)

### Documentation Files
- **Format:** `UPPERCASE_WITH_UNDERSCORES.md` for system files
- **Format:** `Title-Case-With-Hyphens.md` for reference files
- **Format:** `lowercase-with-hyphens.md` for regular files
- **Examples:**
  - ✅ `PROJECT_STATUS.md` (system file)
  - ✅ `How-To-Use-Templates.md` (reference)
  - ✅ `project-structure.md` (regular)
  - ❌ `ERRL_SPEC.md` (inconsistent - should be `ERRL-SPEC.md` or `errl-spec.md`)

### Project Folders
- **Format:** `lowercase-with-hyphens` (preferred)
- **Format:** `lowercase_with_underscores` (acceptable if existing)
- **Examples:**
  - ✅ `errl-portal`
  - ✅ `figma-clone-engine`
  - ✅ `errl_scene_builder` (existing)
